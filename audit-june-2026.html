Most critical issues require less than an hour of work each.

Critical Issues — Must Fix
1
All 18 blog post meta descriptions are too short (65–120 chars, most under 100)
✕ Critical · SEO
Affected Files
All 18 blog posts in /blog/. Worst: underwater-photography-tips-anilao.html at 65 chars. Only pygmy-seahorses-anilao.html passes at 120 chars.
Impact
Google uses these as search result snippets. Under-100-char descriptions force Google to auto-generate snippets from body text — which are typically worse for CTR. This is the single largest SEO gap on the site.
Fix
Rewrite each to 120–155 chars with primary keyword + concrete benefit + CTA. Example: "Identify nudibranchs..." (62 chars) → "Spot 80+ nudibranch species at Anilao dive sites — ID tips, best sites & dive conditions for macro photographers in Batangas." (125 chars)
2
Skip links missing on 5 of 6 pages (WCAG 2.4.1 Level A failure)
✕ Critical · Accessibility
Affected Files
rooms.htmldive.htmlgallery.htmllocation.htmlblog.html
Impact
Keyboard-only users must tab through 9 nav elements before reaching page content on every page load. WCAG 2.4.1 Level A — the most severe category. index.html already has the .skip-link CSS in style.css.
Fix
Add <a href="#main-content" class="skip-link">Skip to main content</a> after <body> on each page. Add id="main-content" to the <main> element. 2-line fix per page.
3
Clickable <div> elements not keyboard accessible (WCAG 2.1.1 Level A failure)
✕ Critical · Accessibility
Affected Files
dive.html (.eco-card species grid)
gallery.html (.uw-card photo grid)
Impact
<div> elements with click handlers cannot receive Tab focus, cannot be activated by Enter/Space, and are invisible to screen readers. Keyboard users cannot access the species modal or the photo modal.
Fix
Add role="button" tabindex="0" to each .eco-card and .uw-card, plus a keydown handler: if (e.key === 'Enter' || e.key === ' ') card.click(). Or convert to <button> elements (preferred).
4
Image filenames with spaces, capitals, and special characters — silent 404 risk on GitHub Pages
✕ Critical · Technical
Affected Files
images/Nature Folder/images/Hero Image/images/Home Page Image Gallery/images/rooms/01. Standard Double.../ and more
Impact
GitHub Pages runs on Linux — case-sensitive. Filenames like "Bargibant's pygmy seahorse (Hippocampus bargibanti).jpg" contain apostrophes, parentheses, and capitals. A single capitalization mismatch = silent 404 in production.
Fix
Batch-rename all image files and directories to lowercase-with-hyphens. Run a script to find all HTML/CSS references and update them to match. This is the only FAIL in the technical audit.
5
preload="auto" on 17 MB and 27 MB hero videos forces full download on page load
✕ Critical · Performance
Affected Files
dive.html (dive-hero.mp4 — 17.5 MB)
gallery.html (gallery-hero.mp4 — 27.2 MB)
Impact
preload="auto" downloads the entire video before the user interacts. On mobile, that's a 17–27 MB download triggered automatically. Causes severe LCP and TTI penalties and likely fails Core Web Vitals on mobile.
Fix
Change preload="auto" to preload="none" on both <video> elements. The browser will still autoplay (autoplay, muted, playsinline, loop, and poster are already set correctly). 1-word change per file.
6
404.html is a silent instant redirect — not a real error page
✕ Critical · Content & UX
Affected Files
404.html
Impact
Broken links redirect silently to homepage. Users following dead bookmarks get no explanation. Google Search Console will report soft 404s — a signal Google uses to assess content quality.
Fix
Build a branded 404 page with nav, an on-brand message ("This cove is hidden"), popular page links (Dive, Rooms, Blog, Location), and a Book Now CTA. Remove the meta refresh and JS redirect. Keep <meta name="robots" content="noindex">.
Warnings — Should Fix
W1
dive.html & gallery.html meta descriptions over 155 chars (191 and 180)
⚠ SEO
Google truncates at ~155 chars, cutting off the CTA on dive.html. Trim to under 155 chars preserving keyword and CTA.
W2
og:site_name missing on index.html (present on all other pages)
⚠ SEO
Homepage renders without site name on Facebook/WhatsApp previews. Add <meta property="og:site_name" content="Casa Escondida Anilao" />. One-line fix.
W3
H1 keyword weakness on index.html, gallery.html, location.html
⚠ SEO
Desktop H1 on homepage is "Casa Escondida" only. Gallery H1 is "Life at Casa Escondida". Location H1 is atmospheric ("A Hidden Cove in Batangas Bay"). All miss primary keyword opportunities.
W4
Room count inconsistency: 31 (meta) vs 24 (on-page stat)
⚠ Content rooms.html
Meta description says "31 ocean-side rooms" but the hero stat says "24 Rooms & Suites" and schema only lists 3 room types. Factual inconsistency visible in search results.
W5
Gold color fails WCAG 4.5:1 contrast at small text in day theme
⚠ Accessibility
#b8862a on #f5f0ea = ~3.1:1 contrast ratio. WCAG 1.4.3 requires 4.5:1. Darken to #7a5a18 for day theme. Night theme gold passes (11.7:1) — no change needed there.
W6
outline:none on form inputs without focus replacement
⚠ Accessibility
index.html (.b-input), location.html (.cf-input, .cf-textarea), blog.html (.sort-select). Add :focus-visible { outline: 2px solid var(--accent); outline-offset: 2px } to all form controls.
W7
rooms.html heading hierarchy skips levels (div → H4)
⚠ Accessibility
Room names (.rm-name) are <div> elements, not H3. Inclusions strip uses H4 with no H2/H3 parent. Breaks semantic outline for screen readers (WCAG 1.3.1).
W8
Booking input at 0.85rem triggers iOS Safari auto-zoom
⚠ Mobile
iOS Safari zooms viewport on any input below 16px. Booking form is the #1 conversion point — jarring zoom on tap hurts UX. Fix: .b-input { font-size: 1rem }. Visual change is minimal.
W9
3 touch targets below 44px (.mob-close, .soc icons, .rm-gallery-btn)
⚠ Mobile
.mob-close: 38×38px, .soc footer icons: 36×36px, .rm-gallery-btn carousel arrows: 30×30px. Add min-width: 44px; min-height: 44px to all three.
W10
Nav not transparent on sub-pages — style.css conflict
⚠ Technical
style.css applies backdrop-filter: blur() to base #nav — not just #nav.scrolled. Nav loads with frosted-glass on rooms, dive, gallery, location, blog. Move background + backdrop-filter to #nav.scrolled only.
W11
site.js theme ID mismatch on index.html (themeBtn vs themeToggle)
⚠ Technical
site.js queries getElementById('themeBtn') but index.html uses id="themeToggle". Works only because index.html has its own inline handler. If cleaned up, theme toggle breaks on homepage.
W12
CSS variable naming divergence: index.html vs style.css
⚠ Technical
index.html uses --bg-primary, --text-primary; style.css uses --bg, --text. Latent maintenance bug — any shared component referencing --bg-primary will silently fail on sub-pages.
W13
Phone and email absent from footer — missing on most pages
⚠ Content
Footer has no phone or email. A visitor on rooms.html or dive.html who wants to call must navigate to location.html. Two lines in footer-v3.css would fix this on every page.
W14
17+ mosaic JPEGs without WebP + Suiteroom_New.jpg at 478 KB
⚠ Performance
Mosaic gallery images and room cards are uncompressed JPEGs. Suiteroom_New.jpg at 478 KB is referenced twice. WebP typically saves 25–35%. Wrap in <picture> elements with WebP source.
W15
Rating pills (4.5★ TripAdvisor) hidden on mobile above the fold
⚠ Mobile / UX
display:none at max-width:768px. Mobile users — likely the majority — see no social proof above the fold. Reformat for mobile (smaller, stacked) instead of hiding entirely.
W16
Blog posts don't cross-link to each other (topical cluster gap)
⚠ SEO / Content
"Best Dive Sites" doesn't link to "Kirby's Rock". "Night Diving" doesn't link to "What to Pack". Internal cross-links build topical cluster authority signals for Google.
